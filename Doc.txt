Any new modules should include in their .csproj the following in order to be able to move the .dll to Host/bin and be detectable

<Target Name="DeployModule" AfterTargets="Build">
  <Copy
    SourceFiles="$(OutputPath)$(AssemblyName).dll"
    DestinationFolder="$(MSBuildThisFileDirectory)../../Host/bin/$(Configuration)/net10.0/modules/" />
</Target>

Modules should reference Core package to be able to implemente IModule:

> dotnet add reference ../../Core/Core.csproj

or add to .csproj:

<ItemGroup>
  <ProjectReference Include="../../Core/Core.csproj" />
</ItemGroup>

Every module should have at least one implementation of IModule, and that will be the entry point from Host to such moduloe
